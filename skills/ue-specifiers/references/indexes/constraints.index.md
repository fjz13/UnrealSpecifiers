# Constraints Index

> Compact guardrail index for specifier/meta combinations while writing UE C++ code. Use this before opening source docs when a declaration combines Blueprint, Details Panel, config, SaveGame, replication, instancing, UMG binding, node shaping, numeric UI, bitmask, or type-picker behavior.
> `Doc` is relative to `references/sources/`.

| Concern | Constraint | Prefer | Doc |
|---|---|---|---|
| Details Panel edit/visibility | Use exactly one of `EditAnywhere`, `EditDefaultsOnly`, `EditInstanceOnly`, `VisibleAnywhere`, `VisibleDefaultsOnly`, or `VisibleInstanceOnly` on a property. | Pick the narrowest edit or visibility scope required by the code path. | `Specifier/UPROPERTY/DetailsPanel/EditAnywhere.md` |
| Details Panel visible-only state | `Visible*` means visible in Details Panel but not editable. It does not grant Blueprint access. | Pair with `BlueprintReadOnly` only when Blueprint reads the value. | `Specifier/UPROPERTY/DetailsPanel/VisibleAnywhere.md` |
| Blueprint property access | `BlueprintReadOnly` and `BlueprintReadWrite` are alternatives for Blueprint access level. | Prefer `BlueprintReadOnly` unless Blueprint mutation is part of the workflow. | `Specifier/UPROPERTY/Blueprint/BlueprintReadOnly.md` |
| Blueprint property accessors | `BlueprintGetter`, `BlueprintSetter`, `Getter`, and `Setter` require matching functions and should not be invented as decoration. | Use accessors only when custom read/write logic exists. | `Specifier/UPROPERTY/Blueprint/BlueprintGetter.md` |
| Blueprint callable function | `BlueprintCallable` exposes a function as an executable Blueprint node; it is not needed for C++-only helpers. | Keep helpers C++-only unless Blueprint callers are required. | `Specifier/UFUNCTION/Blueprint/BlueprintCallable.md` |
| Blueprint pure function | `BlueprintPure` should be side-effect-free and value-query oriented. | Use `BlueprintCallable` for commands or mutating functions. | `Specifier/UFUNCTION/Blueprint/BlueprintPure.md` |
| Category metadata/specifier | `Category` organizes editor/Blueprint presentation; it is not a behavior, persistence, or access-control specifier. | Add only for exposed editor or Blueprint API organization. | `Specifier/UPROPERTY/DetailsPanel/Category.md` |
| Conditional Details Panel UI | `EditCondition`, `EditConditionHides`, `InlineEditConditionToggle`, and `HideEditConditionToggle` affect editor presentation only. | Use C++ validation for runtime invariants; use conditional UI only for editor ergonomics. | `Meta/DetailsPanel/EditCondition.md` |
| Numeric editor limits | `ClampMin`/`ClampMax` are hard editor input clamps; `UIMin`/`UIMax` are slider/display ranges. | Use clamp metadata for editor input bounds and still validate runtime values where needed. | `Meta/Numeric/ClampMin.md` |
| String editor UI | `MultiLine`, `MaxLength`, and `PasswordField` are editor UI hints/limits, not security or runtime validation. | Keep security-sensitive data out of ordinary reflected string properties. | `Meta/String/PasswordField.md` |
| SaveGame persistence | `SaveGame` only marks fields for SaveGame serialization flows; it does not save automatically by itself. | Use only for fields intentionally serialized by a SaveGame object/path. | `Specifier/UPROPERTY/Serialization/SaveGame.md` |
| Transient state | `Transient` is the opposite of persistence intent and should not be used as a convenience default for normal editable data. | Use it only for values rebuilt after load or runtime-only caches. | `Specifier/UPROPERTY/Serialization/Transient.md` |
| Config property | `Config`/`GlobalConfig` require a config-aware owner class setup and are not runtime persistence shortcuts. | Use config only for ini-backed settings. | `Specifier/UPROPERTY/Config/Config.md` |
| Config class destination | `DefaultConfig`, `GlobalUserConfig`, and `ProjectUserConfig` are class-side config destination choices; they do not make a property config-backed by themselves. | Put `Config=...` on the class and `Config`/`GlobalConfig` on the properties that should load/save. | `Specifier/UCLASS/Config/DefaultConfig.md` |
| Replicated property | `Replicated` needs the normal UE replication setup, including lifetime registration where required. | Use only for authoritative network state. | `Specifier/UPROPERTY/Network/Replicated.md` |
| RepNotify property | `ReplicatedUsing` requires a valid notify function and should match the property update semantics. | Use when clients need a notification hook for replicated changes. | `Specifier/UPROPERTY/Network/ReplicatedUsing.md` |
| Replicated struct member exclusion | Property-side `NotReplicated` is for USTRUCT members inside replicated structs, not arbitrary actor properties. | Leave non-replicated actor properties unmarked; use `NotReplicated` only to opt a struct member out of its containing replicated struct. | `Specifier/UPROPERTY/Network/NotReplicated.md` |
| RPC direction | `Server`, `Client`, and `NetMulticast` are RPC direction specifiers, not ordinary function exposure flags. | Use only for network calls with clear authority/ownership semantics. | `Specifier/UFUNCTION/Network/Server.md` |
| RPC reliability | `Reliable`/`Unreliable` apply to RPC delivery behavior and should not be added to non-RPC functions. | Prefer `Unreliable` for frequent cosmetic updates; reserve `Reliable` for critical low-frequency events. | `Specifier/UFUNCTION/Network/Reliable.md` |
| RPC validation | `WithValidation` is an RPC validation hook pattern, not a substitute for authority, ownership, rate limiting, or business validation. | Validate server-authoritative inputs explicitly in the server path. | `Specifier/UFUNCTION/Network/WithValidation.md` |
| Blueprint authority/cosmetic filters | `BlueprintAuthorityOnly` and `BlueprintCosmetic` gate local Blueprint execution contexts; they are not RPC send directions. | Use `Server`, `Client`, or `NetMulticast` when the call must cross the network. | `Specifier/UFUNCTION/Network/BlueprintAuthorityOnly.md` |
| Instanced subobject property | Property-side `Instanced` is for owned inline object references and usually needs compatible class-side creation/editing support. | Pair with class-side `EditInlineNew` when editor inline creation is required. | `Specifier/UPROPERTY/Instanced.md` |
| Inline-editable class | `EditInlineNew` belongs on the UObject class that may be created inline, not on the property. | Use with property-side `Instanced` for editor-created subobjects. | `Specifier/UCLASS/Instance/EditInlineNew.md` |
| Default instancing | `DefaultToInstanced` is class-side default instancing behavior; it does not replace property-side ownership review. | Prefer property-side `Instanced` for narrow ownership; use class-side defaults only when the type generally behaves as an inline subobject. | `Specifier/UCLASS/Instance/DefaultToInstanced.md` |
| Inline Details Panel display | `meta=(EditInline)` and `meta=(NoEditInline)` affect Details Panel inline display; they do not create owned subobject semantics by themselves. | Use `Instanced` plus class-side `EditInlineNew` for owned inline UObject creation/editing. | `Meta/DetailsPanel/EditInline.md` |
| UMG widget binding | `BindWidget` and `BindWidgetAnim` require matching Widget Blueprint names/types and are not dynamic lookup helpers. | Use optional binding or explicit runtime lookup when the widget/animation may be absent. | `Meta/Widget/BindWidget.md` |
| UMG animation binding | `BindWidgetAnim` properties must be `Transient` and usually `UWidgetAnimation*`; otherwise Widget Blueprint binding reports an error. | Add `Transient` with `meta=(BindWidgetAnim)` or use the optional form when the animation may be absent. | `Meta/Widget/BindWidgetAnim.md` |
| World context metadata | `meta=(WorldContext="ParamName")` must name an existing parameter, usually `UObject*` or `const UObject*`. | Use primarily for Blueprint function library/static helper APIs that need world resolution. | `Meta/Blueprint/WorldContext.md` |
| World context modifiers | `CallableWithoutWorldContext` and `ShowWorldContextPin` modify callability/pin display; they do not replace `WorldContext`. | Use only after confirming the call context behavior. | `Meta/Blueprint/CallableWithoutWorldContext.md` |
| Blueprint event implementation | `BlueprintImplementableEvent` means Blueprint supplies the implementation; `BlueprintNativeEvent` needs a C++ `_Implementation`. | Use NativeEvent when C++ default behavior must exist. | `Specifier/UFUNCTION/Blueprint/BlueprintNativeEvent.md` |
| Blueprint event override shape | `ForceAsFunction` changes how a Blueprint event override is presented; output or return parameters already force function-style override behavior. | Add `ForceAsFunction` only when a no-output Blueprint event should appear as an overrideable function. | `Meta/Blueprint/ForceAsFunction.md` |
| Editor callable function | `CallInEditor` is for editor Details Panel buttons and should not be treated as runtime gameplay exposure. | Use editor tools for complex editor workflows; use `BlueprintCallable` for runtime Blueprint calls. | `Specifier/UFUNCTION/Blueprint/CallInEditor.md` |
| Blueprint node shaping | `AdvancedDisplay`, `DefaultToSelf`, `AutoCreateRefTerm`, `ExpandBoolAsExecs`, and `ExpandEnumAsExecs` change node usability, not C++ behavior. | Add only when the Blueprint node shape clearly improves call-site correctness. | `Meta/Blueprint/AdvancedDisplay.md` |
| Self/default pin shaping | `DefaultToSelf` gives a named object parameter a self default; `HideSelfPin` hides the member-function target pin, not arbitrary static function parameters. | Use `HidePin` or `InternalUseParam` to hide a `DefaultToSelf` parameter on a static Blueprint library function. | `Meta/Pin/HideSelfPin.md` |
| Latent Blueprint function | `Latent` functions need the latent function pattern, not just async C++ code. | Use delegates/events for ordinary async notification. | `Meta/Blueprint/Latent/Latent.md` |
| Latent info parameter | A `Latent` Blueprint function should name its `FLatentActionInfo` parameter with `LatentInfo`; otherwise the Blueprint VM cannot fill the latent continuation data correctly. | Add `meta=(Latent, LatentInfo="LatentInfo")` and a matching `FLatentActionInfo LatentInfo` parameter. | `Meta/Blueprint/Latent/LatentInfo.md` |
| Custom thunk API | `CustomThunk` and `CustomStructureParam` are advanced manual thunk paths with high maintenance cost. | Avoid unless normal UHT-generated Blueprint calls cannot express the API. | `Meta/UHT/CustomThunk.md` |
| UPARAM direction/display | `UPARAM(ref)` changes Blueprint pin direction interpretation; `UPARAM(DisplayName=...)` only changes pin text. | Use return values or explicit output refs for output data; use display names sparingly. | `Specifier/UPARAM/Blueprint/ref.md` |
| Enum bitmask UI | `Bitflags`, `Bitmask`, and `BitmaskEnum` require bitmask semantics; they are not ordinary enum dropdown metadata. | Use plain `BlueprintType` enum for single-choice states. | `Meta/Enum/Bitmask.md` |
| Type picker narrowing | Picker metadata should narrow allowed classes, structs, interfaces, or assets instead of accepting broad object/class choices. | Use the narrowest valid filter such as `AllowedClasses`, `MustImplement`, `MetaClass`, or `BaseStruct`. | `Meta/TypePicker/AllowedClasses.md` |
| Blueprint type exposure | `BlueprintType`, `Blueprintable`, `NotBlueprintType`, and `NotBlueprintable` solve different exposure problems. | Choose variable/parameter exposure separately from Blueprint inheritance exposure. | `Specifier/UCLASS/Blueprint/BlueprintType.md` |
| Deprecated or removed items | Deprecated/removed specifiers and metadata are for diagnosis or migration, not new code. | Prefer current UE5.8-supported behavior. | `Specifier/UPROPERTY/Network/RepRetry.md` |
